🎯 **What:**
Fixed an insecure HTTP vulnerability in the route planner service that transmitted external API requests in plaintext.

⚠️ **Risk:**
Using `http://` for external API requests allows for Man-in-the-Middle (MITM) attacks. An attacker could potentially intercept and modify the route requests, seeing the unencrypted location coordinates, or tamper with the routing responses.

🛡️ **Solution:**
Updated the external API URL for `router.project-osrm.org` to use the secure `https://` protocol instead of `http://`, ensuring that location coordinates and routing responses are encrypted in transit.
