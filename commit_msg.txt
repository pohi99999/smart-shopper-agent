🔒 Fix insecure path traversal in prices data lookup

🎯 What:
Replaced hardcoded and insecure relative path fallbacks (`os.Stat("../../internal/data/prices.json")`) in `getPricesFilePath()` with the secure `PRICES_FILE_PATH` environment variable.

⚠️ Risk:
The previous implementation allowed arbitrary relative path traversal and relied on the application's current working directory, which can vary across deployments and potentially expose or manipulate unintended files depending on deployment structure.

🛡️ Solution:
Updated `getPricesFilePath` to fetch the path from `PRICES_FILE_PATH`, defaulting safely to `internal/data/prices.json` if unset. Updated all associated test functions to use `t.TempDir()` and override the environment variable, safely eliminating side-effects that previously modified real repository files.
